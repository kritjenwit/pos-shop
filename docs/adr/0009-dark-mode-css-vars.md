# ADR-0009: Dark Mode via CSS Custom Properties

Replaced all hardcoded color references with CSS custom properties and introduced a `ThemeContext` for dark mode toggling. The `COLORS` constants object now maps to `var(--color-*)` CSS variables instead of raw hex values. `index.css` defines both `:root` (light) and `.dark` (dark) variable sets. `ThemeContext` persists the user's preference to `localStorage` key `pos-shop-theme` and respects `prefers-color-scheme` on first visit. A Moon/Sun toggle button in `StaffLayout` header switches the theme at runtime via `document.documentElement.classList.toggle('dark')`. Tailwind configured with `darkMode: 'class'` for compatibility. Twelve alpha variants (e.g., `--color-primary-04` through `--color-primary-80`) were pre-computed for the eight colors that use alpha blending — avoiding runtime `rgba()` calls that would break the CSS var approach.
